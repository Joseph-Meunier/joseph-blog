## C'est quoi la programmation orientée objet?

La programmation orientée objet (POO) est un paradigme de programmation basé sur le concept d'"objets", qui peuvent contenir des données et du code pour manipuler ces données.

Prenons l'exemple d'un animal.

Avant, si nous n'utilisions pas la POO, nous aurions du code comme ceci :

```csharp
var animalName = "John";
var animalAge = 2;

var animalName = "Jane";
var animalAge = 7;
```

Mais maintenant, si nous utilisons la POO, nous aurions du code comme ceci :

```csharp
var animal = new Animal();
animal.Name = "John";
animal.Age = 10;

var animal2 = new Animal();
animal2.Name = "Jane";
animal2.Age = 6;
```

Cela nous permet de créer des objets qui peuvent être instanciés et avoir des propriétés et des méthodes.

### Classe

Une classe est un plan pour créer des objets.
Elle définit les propriétés et les méthodes d'un objet.

```csharp
class Animal {
    public string Name { get; set; }
    public int Age { get; set; }
}
```

### Objet

Un objet est une instance d'une classe.
Il possède les propriétés et les méthodes de la classe.

```csharp
var animal = new Animal();
```

### Instanciation

L'instanciation est le processus de création d'un objet à partir d'une classe.

```csharp
var animal = new Animal();
```

### Propriétés

Les propriétés sont des variables qui appartiennent à un objet.
Dans notre exemple, les propriétés sont `Name` et `Age`.

```csharp
animal.Name = "John";
animal.Age = 10;
```

### Méthodes

Les méthodes sont des fonctions qui appartiennent à un objet.
Dans notre exemple, nous pouvons ajouter les méthodes `Eat`, `Sleep`, et `MakeSound`.
Nous pouvons aussi voir que nous pouvons accéder aux propriétés de l'objet dans les méthodes.

```csharp
class Animal {
    private string name;
    private int age;

    public void Eat() {
        Console.WriteLine($"{name} mange");
    }

    public void Sleep() {
        Console.WriteLine($"{name} dort");
    }

    public void MakeSound() {
        Console.WriteLine($"{name} fait un son");
    }
}
```

## Les 3 principes de la POO

En POO, nous avons 3 principes :

1. Encapsulation
2. Héritage
3. Polymorphisme

### Encapsulation

L'encapsulation est le principe d'encapsuler les données et les méthodes d'un objet.
Cela nous permet de protéger les données et les méthodes d'un objet en les rendant privées.
Cela rend aussi les données et les méthodes d'un objet plus faciles à maintenir et à tester.

Dans notre exemple, nous avons encapsulé les données et les méthodes d'un objet.
Les propriétés sont privées et les méthodes sont publiques.

```csharp
class Animal {
    private string name;
    private int age;

    public void Eat() {
        Console.WriteLine($"{name} mange");
    }

    public void Sleep() {
        Console.WriteLine($"{name} dort");
    }

    public void MakeSound() {
        Console.WriteLine($"{name} fait un son");
    }
}

var animal = new Animal();
animal.Name = "John";
animal.Age = 10;

animal.Eat();
animal.Sleep();
animal.MakeSound();
```

Si nous l'exécutons, nous obtenons le résultat suivant :
```csharp
"John mange"
"John dort"
"John fait un son"
```

### Héritage

L'héritage est le principe d'hériter des données et des méthodes d'un objet.
Cela nous permet de créer une classe qui hérite des propriétés et des méthodes d'une autre classe.
Cela nous permet de réduire la duplication de code et de rendre le code plus facile à maintenir, à lire et à tester.

Si nous reprenons notre exemple, nous pouvons créer une classe `Dog` et une classe `Cat` qui héritent de la classe `Animal`.

```csharp
class Dog : Animal {
    public override void MakeSound() {
        Console.WriteLine($"{name} aboie");
    }
}

class Cat : Animal {
    public override void MakeSound() {
        Console.WriteLine($"{name} miaule");
    }
}

```
We can see that the `Dog` class and the `Cat` class inherit from the `Animal` class and that the `MakeSound` method is redefined (it is overridden) in the `Dog` and `Cat` classes.

### Polymorphisme

Le polymorphisme est le principe de polymorphiser les données et les méthodes d'un objet.
Cela signifie que nous pouvons utiliser les mêmes méthodes pour différents objets.

Si nous reprenons notre exemple, nous pouvons créer une classe `Dog` et une classe `Cat` qui héritent de la classe `Animal` et nous pouvons utiliser les mêmes méthodes pour différents objets.
Ci-dessous, nous créons un objet `animal` de la classe `Animal` et nous appelons la méthode `MakeSound` qui est définie dans la classe `Animal`.
Ensuite, nous créons un objet `dog` de la classe `Dog` et nous appelons la méthode `MakeSound` qui est définie dans la classe `Dog`.
Enfin, nous créons un objet `cat` de la classe `Cat` et nous appelons la méthode `MakeSound` qui est définie dans la classe `Cat`.

```csharp
var animal = new Animal();
animal.Name = "John";
animal.MakeSound();

var dog = new Dog();
dog.Name = "Rex";
dog.MakeSound();

var cat = new Cat();
cat.Name = "Whiskers";
cat.MakeSound();
```

Si nous l'exécutons, nous obtenons le résultat suivant :

```csharp
"John fait un son"
"Rex aboie"
"Whiskers miaule"
```

## Abstraction

Abstract est un mot-clé qui nous permet de créer une classe qui ne peut pas être instanciée.

Nous pouvons aussi voir que la classe `Animal` n'a pas vraiment besoin d'être instanciée.
Un animal est nécessairement d'une espèce.
Pour empêcher l'instanciation de la classe `Animal`, nous pouvons la rendre abstraite.

```csharp
abstract class Animal {
    public string Name { get; set; }
    public int Age { get; set; }
}

```

## Conclusion

La POO est un paradigme de programmation qui nous permet de créer des objets qui ont des propriétés et des méthodes.
Cela rend le code plus facile à maintenir, à lire et à tester.
