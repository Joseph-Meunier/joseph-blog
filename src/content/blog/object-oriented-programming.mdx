## C'est quoi la programmation orientée objet?

La programmation orientée objet (POO) est un paradigme de programmation basé sur le concept d'"objets", qui peuvent contenir des données et du code pour manipuler ces données.

Prenons l'exemple d'un animal.

Avant, si nous n'utilisions pas la POO, nous aurions du code comme ceci :

```csharp
var animal1Name = "John";
var animal1Age = 2;

var animal2Name = "Jane";
var animal2Age = 7;
```

Mais maintenant, si nous utilisons la POO, nous aurions du code comme ceci :

```csharp
var animal1 = new Animal();
animal1.Name = "John";
animal1.Age = 10;

var animal2 = new Animal();
animal2.Name = "Jane";
animal2.Age = 6;
```

Cela nous permet de créer des objets qui peuvent être instanciés et avoir des propriétés et des méthodes.

## Classe

Une classe est un plan pour créer des objets.
Elle définit les propriétés et les méthodes d'un objet.

```csharp
class Animal {
    public string Name { get; set; }
    public int Age { get; set; }
}
```

## Objet

Un objet est une instance d'une classe.
Il possède les propriétés et les méthodes de la classe.

```csharp
var animal = new Animal();
```

## Instanciation

L'instanciation est le processus de création d'un objet à partir d'une classe.

```csharp
var animal = new Animal();
```

## Propriétés

Les propriétés sont des variables qui appartiennent à un objet.
Dans notre exemple, les propriétés sont `Name` et `Age`.

```csharp
animal.Name = "John";
animal.Age = 10;
```

## Méthodes

Les méthodes sont des fonctions qui appartiennent à un objet.
Dans notre exemple, nous pouvons ajouter les méthodes `Eat`, `Sleep`, et `MakeSound`.
Nous pouvons aussi voir que nous pouvons accéder aux propriétés de l'objet dans les méthodes.

```csharp
class Animal {
    private string Name;
    private int Age;

    public void Eat() {
        Console.WriteLine($"{this.Name} mange");
    }

    public void Sleep() {
        Console.WriteLine($"{this.Name} dort");
    }

    public void MakeSound() {
        Console.WriteLine($"{this.Name} fait un son");
    }
}
```

NB: 
`this` est une référence à l'objet en cours.
Il est utilisé pour accéder aux propriétés et aux méthodes de l'objet en cours.
Dans notre exemple, nous avons utilisé `this.Name` pour accéder à la propriété `Name` de l'objet en cours.
A noter que `this` est optionnel dans la majorité des cas, sauf en cas d'ambiguïté (ex: nom de variable identique).

## Les 4 principes de la POO

En POO, nous avons 4 principes :

1. Encapsulation
2. Héritage
3. Polymorphisme
4. Abstraction

### Encapsulation

L'encapsulation est le principe d'encapsuler les données et les méthodes d'un objet.
On masque l’implémentation interne et expose des points d’accès contrôlés (ex: propriétés avec logique dans les get ou set)
Cela rend aussi les données et les méthodes d'un objet plus faciles à maintenir et à tester.

Dans notre exemple, nous avons encapsulé les données de l'objet en les rendant privées, et nous exposons des méthodes publiques pour interagir avec elles.
Les données sont rendues privées et accessibles via des propriétés ou des méthodes publiques, permettant un contrôle sur la lecture/écriture.

```csharp
class Animal {
    private string _name; // Champs privés
    private int _age; // Champs privés

    public Animal(string name, int age) {
        _name = name;
        _age = age;
    }

    public void Eat() {
        Console.WriteLine($"{_name} mange");
    }

    public void Sleep() {
        Console.WriteLine($"{_name} dort");
    }

    public void MakeSound() {
        Console.WriteLine($"{_name} fait un son");
    }
}


var animal = new Animal("John", 10);

animal.Eat();
animal.Sleep();
animal.MakeSound();
```

NB:
Nous avons utilisé un constructeur pour initialiser les champs privés de l'objet.
Le constructeur est une méthode qui est appelée lors de l'instanciation d'un objet.
Dans notre exemple, nous avons utilisé un constructeur pour initialiser les champs privés `_name` et `_age` de l'objet.
```csharp
    public Animal(string name, int age) {
        _name = name;
        _age = age;
    }
```

Si nous l'exécutons, nous obtenons le résultat suivant :
```csharp
"John mange"
"John dort"
"John fait un son"
```

NB:
Nous avons utilisé des getters et setters pour utiliser les propriétés `Name` dans notre cas, pour l'utiliser dans la méthode `MakeSound`.
Getters et setters sont des méthodes qui permettent d'accéder aux propriétés de l'objet en cours.
Getters sont utilisés pour récupérer la valeur d'une propriété.
Setters sont utilisés pour modifier la valeur d'une propriété.

### Héritage

L'héritage est le principe d'hériter des données et des méthodes d'un objet.
Cela nous permet de créer une classe qui hérite des propriétés et des méthodes d'une autre classe.
Cela nous permet de réduire la duplication de code et de rendre le code plus facile à maintenir, à lire et à tester.

Si nous reprenons notre exemple, nous pouvons créer une classe `Dog` et une classe `Cat` qui héritent de la classe `Animal`.

```csharp
class Animal {
    public string Name { get; protected set; }
    public abstract void MakeSound();
}

class Dog : Animal {
    public override void MakeSound() {
        Console.WriteLine($"{this.Name} aboie");
    }
}

class Cat : Animal {
    public override void MakeSound() {
        Console.WriteLine($"{this.Name} miaule");
    }
}

```
On peut voir que la classe Dog et la classe Cat héritent de la classe Animal, et que la méthode MakeSound est redéfinie (override) dans ces classes.
A noter aussi que la propriété `Name` est protégée (protected), ce qui signifie que les classes qui héritent de la classe `Animal` peuvent accéder à la propriété `Name`, mais pas à la modifier.

### Polymorphisme
Le polymorphisme permet d'utiliser une interface ou une méthode commune pour manipuler des objets de types différents (ayant un comportement spécifique).
Exemple : appeler .MakeSound() sur un Animal, un Dog, ou un Cat, et obtenir un résultat différent.

Si nous reprenons notre exemple, nous pouvons créer une classe `Dog` et une classe `Cat` qui héritent de la classe `Animal` 
et nous pouvons utiliser les mêmes méthodes pour différents objets.
Ensuite, nous créons un objet `dog` de la classe `Dog` et nous appelons la méthode `MakeSound` qui est définie dans la classe `Dog`.
Enfin, nous créons un objet `cat` de la classe `Cat` et nous appelons la méthode `MakeSound` qui est définie dans la classe `Cat`.
Et nous les stockons dans une liste d'animaux.

```csharp
abstract class Animal {
    public string Name { get; set; }
    public abstract void MakeSound();
}

class Dog : Animal {
    public override void MakeSound() {
        Console.WriteLine($"{Name} aboie");
    }
}

class Cat : Animal {
    public override void MakeSound() {
        Console.WriteLine($"{Name} miaule");
    }
}

// Exemple d'utilisation
var animals = new List<Animal>
{
    new Dog { Name = "Rex" },
    new Cat { Name = "Mimi" }
};

foreach (var animal in animals)
{
    animal.MakeSound(); // Appelle la bonne version selon le type réel
}
```

Si nous l'exécutons, nous obtenons le résultat suivant :

```csharp
"Rex aboie"
"Mimi miaule"
```

### Abstraction

Le mot clé abstract permet de créer une classe qui ne peut pas être instanciée.

Les classes qui héritent d'une classe abstraite doivent obligatoirement implémenter ses méthodes abstraites.

Nous pouvons aussi voir que la classe `Animal` n'a pas vraiment besoin d'être instanciée.
Un animal est nécessairement d'une espèce.
Pour empêcher l'instanciation de la classe `Animal`, nous pouvons donc la rendre abstraite.

```csharp
abstract class Animal {
    public string Name { get; set; }
    public int Age { get; set; }
    public abstract void MakeSound();
}
class Dog : Animal {
    public override void MakeSound() {
        Console.WriteLine($"{Name} aboie");
    }
}
```

## Conclusion

La POO est un paradigme de programmation qui nous permet de créer des objets qui ont des propriétés et des méthodes.
Cela rend le code plus facile à maintenir, à lire et à tester.
